# 英语趣学 H5 · 产品需求文档（PRD v0.2）

> **更新记录**：v0.1 → v0.2 基于 4 项关键决策收敛——商业模式=**免费+广告**；联想定位=**趣味须准确**；复习机制=**间隔重复算法**；端侧重=**移动为主**。据此修订范围、指标、内容规范、技术、合规与路线图。

## 1. 背景与问题（Why）
**用户问题**：市面背单词产品普遍"机械重复、缺乏联想、不好玩"，尤其全龄用户难以坚持。
**机会点**：用「图像故事联想」做记忆钩子 + 「闯关积分排行」做留存引擎 + 「自适应分级」兜住全龄。
**商业背景（v0.2 新增）**：采用**免费 + 广告**模式 → 用户时长即收入，因此**留存引擎（间隔重复 + 游戏化）与广告库存强绑定**，产品目标与变现目标一致。

## 2. 目标用户与场景
- **全龄自适应**：靠分级测试定档，覆盖小学生到备考成人。
- **移动为主（v0.2）**：核心场景是通勤、排队、睡前等碎片 5–10 分钟单手学习；PC 端"能学但非重点"，作为长尾兼容。
- 三类典型画像（同一定级框架下的不同档位）：
  - 小学生（L1–L2，需家长视角/适龄）
  - 成人零基础（L1–L2，兴趣驱动）
  - 备考成人（L3–L4，目标导向）

## 3. 核心假设与待验证（Riskiest Assumptions）
- **A1**：图像故事联想（须准确）相较普通释义，记忆留存提升 ≥15% → 用 A/B 验证（MVP 用零成本 Emoji+模板故事先灰度）。
- **A2**：间隔重复算法显著提升 7 日 / 30 日留存。
- **A3**：免费 + 广告可被全龄（含未成年）接受，且不显著拉低留存（设广告疲劳监控）。
- **A4**：移动为主下 PC 长尾可用，不丢关键场景。
- **A5**：「趣味须准确」的联想内容可被低成本规模化生产且过审。

## 4. 产品目标与成功指标
- **北极星指标**：周完成「学习 → 联想 → 测验 → 入复习队列」完整闭环的活跃用户数（**WAL, Weekly Active Learners**）。比 DAU 更反映真实学习价值，避免"只打卡不学"。
- **驱动指标**：
  - 新用户分级测试完成率（目标 ≥80%）
  - 人均日学习词数
  - 关卡通过率 / 平均连续打卡天数（streak）
  - 复习任务完成率
  - 广告可售库存（有效展示频次，feeding 变现）
- **健康指标**：
  - 次日 / 7 日 / 30 日留存
  - 联想「准确且有用」好评率（≥4/5）
  - 低分段用户（排行末 20%）流失率（防劝退）
  - **广告相关流失率**（因广告跳出/卸载）、eCPM、广告疲劳度
  - 复习正确率分布（防太易/太难）
- **基线 & 目标**：埋点后 2–4 周建立基线；目标值（如 WAL 第 8 周 ≥ X）待基线后定。

## 5. 功能范围（In Scope）— MVP
### 5.1 用户与分级（含升降档）
- 游客可用 + 可选注册（云同步进度/跨端）
- 分级测试（10–15 题自适应）定档
- **升降档信号**：连续 N 次测验正确率 >90% 升档，<50% 降档（用户可手动覆盖）

### 5.2 学习核心：单词卡 + 图像故事联想
- 单词卡：词、音标、发音、释义
- 图像：每个词配图（MVP 用 Emoji/图标）
- 故事/谐音：联想记忆文案（**须准确**，见 §10 规范）
- 联想区与「正确拼写/音标」视觉分离，避免误导

### 5.3 学后测验 + 间隔重复队列
- 学后即时测验（选择/听写）
- 测验表现 → 简化 SM-2 计算下次复习时间 → 进入复习队列
- 复习提醒（推送/站内）

### 5.4 游戏化：闯关 + 积分 + 打卡 + 周排行
- 关卡（按词包/单元）、积分、连续打卡（streak）
- **周排行榜（快照式，防性能压力）**

### 5.5 进度与回顾
- 学习地图、错词本、复习日历

### 5.6 商业化（MVP 轻量预留）
- 激励视频：复活 / 提示 / 兑换「去广告体验券」
- 关卡间隙插屏（频次受控）
- 广告位合规（未成年人分流，不投诱导/不适广告）

## 6. 用户故事（标准格式）
- 作为**学习者**，我希望每个单词都有配图和联想小故事，以便更快记住且不枯燥。
- 作为**学习者**，我希望学过的词按科学间隔自动安排复习，以便长期记住而非学过就忘。
- 作为**移动用户**，我希望单手就能完成一张词卡学习，以便在通勤中碎片学习。
- 作为**免费用户**，我希望看一段激励视频换一次提示/复活，以便卡关时不花钱继续（同时支撑变现）。
- 作为**低龄用户家长**，我希望广告内容适龄且不诱导消费，以便放心给孩子用。
- 作为**进阶用户**，我希望系统在我连续做对后自动升档，以便一直学到适合的难度。

## 7. 非目标（Non-goals）
- MVP **不做**：AI 实时对话陪练、社交对战/PK、小程序端、PC 专属重交互、UGC 众包生产联想。
- MVP **仍做**：移动端完整体验 + PC 响应式可用（非重点）。
- 不做自创课程体系，首期用成熟分级词表。

## 8. 信息架构与核心流程（移动优先）
**核心闭环**：
`分级测试定档 → 进入关卡 → 单词卡(图+联想) → 学后测验 → SM-2 入复习队列 → 积分/打卡/排行 → 复习提醒回流`
**页面（移动优先）**：首页(学习地图) / 学习页(词卡) / 测验页 / 复习页 / 我的(进度·错词·排行·设置)

## 9. 难度分级设计
- 4 档（类比 CEFR）：L1 入门 / L2 基础 / L3 进阶 / L4 高阶
- 每档映射词库子集 + 测验难度
- **升降档机制**：正确率信号自动调档 + 手动覆盖，防定档后不匹配

## 10. 联想解释内容策略（关键 · v0.2「须准确」）
- **定位**：趣味但**必须准确**，不误导正确拼写/词义（否决"纯趣味钩子"路线）。
- **三档图像**：① Emoji/图标（MVP 零成本）② 插画图标库 ③ AI 文生图（A1 验证后上）。
- **故事/谐音**：模板化 + AI 生成，经「准确 + 趣味」双校验；人工抽检 + 用户举报。
- **准确性规范**：
  - 联想区与「正确拼写/音标」视觉分离；
  - 联想文案**不得**出现错误拼写引导（如谐音不得替代正字展示）；
  - 敏感词过滤、未成年人适龄。
- **规模与成本**：首期 2000–3000 高频词；AI 文生图按张计费（约 ¥0.02–0.1/张），验证后可控。

## 11. 技术方向建议（尊重工程现实）
- 前端：Vue3 + Vite，**移动优先响应式**；Pinia + Vue Router
- 发音：浏览器 TTS（SpeechSynthesis）+ 可选真人音源（零成本起步）
- 复习：简化 SM-2 调度（本地 + 云端同步）
- 广告：接入广告 SDK（激励视频/插屏），**未成年分流**逻辑
- 后端：轻量 Node / Serverless；用户 / 进度 / 积分 / 排行（周快照）/ 广告事件
- 架构预留：学习逻辑 / 词库 / 用户服务抽成独立 module，小程序端复用相同 API
- 埋点：day 1 全埋（含广告事件）

## 12. 风险与合规（加强未成年 + 广告）
- **未成年人**：隐私政策、年龄门、家长同意（如需收集未成年信息）、广告合规（不得诱导未成年人、限制广告类型/频次）。
- **内容安全**：AI 生成图像/故事需审核，防不当内容。
- **游戏化致瘾**：单日学习上限提醒、护眼。
- **词库版权**：用自有/授权词库，避免直接爬取付费词典。
- **广告伤体验**：A3 验证 + 广告疲劳监控（超阈值降频）。

## 13. 路线图建议
- **MVP（4–6 周）**：游客/注册、分级(含升降档)、单词卡 + 准确联想、学后测验、SM-2 复习、闯关积分打卡、周排行、轻量广告位、错词本。
- **V1.1**：AI 文生图升级联想、复习算法调优、广告变现深化（激励体系 / 去广告券）。
- **V1.2**：社交雏形、PC 体验优化、家长视角。
- **V2**：小程序版（复用 API）、AI 对话陪练（探索）。

## 14. 待决策（已收敛 + 剩余）
**已收敛（本次 4 项决策）**：商业模式=免费+广告；联想=趣味须准确；复习=间隔重复；端=移动为主。
**仍需确认（建议方案已给）**：
1. 词库来源与版权 → 建议：自整理高频词表 2000–3000，避免爬付费词典。
2. 图像预算 → 建议：MVP Emoji/图标，A1 验证后上 AI 文生图。
3. 账号体系 → 建议：游客可用 + 可选注册云同步。
4. 发音 → 建议：TTS 起步。
5. 首期词量/分级覆盖 → 建议 2000–3000 高频词，覆盖 L1–L3。
6. 未成年人年龄门深度 → 是否强制家长同意（影响合规流程与账号设计）。
