# 英语趣学 H5 · 技术方案与开发任务排期

> 依据 PRD v0.2（`英语趣学H5_PRD_v0.md`）产出。身份切换：产品需求已定，本文档为**开发视角**的技术方案与排期。
> 技术栈遵循 PRD 已锁定决策：前端 Vue3 + Vite 移动优先 SPA；后端采用 Laravel 提供 API（与开发主栈一致、稳定），也可替换为 Node/Serverless。

---

## 1. 技术方案

### 1.1 总体架构

```
┌─────────────────────────────────────────────┐
│  H5 客户端（Vue3 + Vite，移动优先 PWA）        │
│  单词卡 / 测验 / 复习 / 闯关 / 排行 / 广告 SDK   │
└───────────────┬─────────────────────────────┘
                │ HTTPS / REST + JWT(Sanctum)
┌───────────────▼─────────────────────────────┐
│  API 网关 / 负载均衡                          │
└───────────────┬─────────────────────────────┘
   ┌────────────┼────────────────┐
   ▼            ▼                ▼
 用户服务     学习核心服务      游戏化 & 广告服务
 （鉴权/分级） （词库/测验/SM-2） （积分/打卡/排行/广告）
   └────────────┼────────────────┘
                ▼
   数据层：MySQL（业务） + Redis（缓存/队列/排行热数据）
   离线：客户端本地队列（IndexedDB）同步复习进度
```

### 1.2 技术栈
- **前端**：Vue 3 + TypeScript + Vite + Pinia + Vue Router；样式 Tailwind CSS（premium 动效）；动画 GSAP / CSS；可选 Three.js 做庆祝/通关特效。
- **移动优先 & PWA**：响应式一套代码；PWA（离线学习、加到主屏），契合"免费+广告"留存目标。
- **后端 API**：Laravel 11 + Sanctum 鉴权 + MySQL + Redis（缓存/队列）。定时任务：周排行榜快照、复习到期计算。
- **发音**：浏览器 SpeechSynthesis（TTS）起步，预留真人音源字段。
- **广告**：接入广告 SDK（如优量汇/穿山甲/AdSense），前端埋点上报，未成年分流。
- **埋点**：自建事件上报端点 + 看板（day 1 全埋，含广告事件）。
- **部署**：前端静态托管（CDN）；后端 Laravel 容器/Serverless。

### 1.3 系统分层与模块
| 层 | 模块 | 职责 |
|----|------|------|
| 表现层 | H5 SPA（PWA） | 单词卡、测验、复习、闯关、排行、广告 |
| 接口层 | API 网关 | 鉴权、限流、版本管理 |
| 业务层 | 用户服务 | 游客/注册、分级测试、升降档 |
| 业务层 | 学习核心 | 词库、单词卡（图+联想）、测验、SM-2 复习调度 |
| 业务层 | 游戏化&广告 | 积分、打卡、关卡进度、周排行快照、广告事件 |
| 数据层 | MySQL + Redis | 持久化 + 缓存/队列 |

### 1.4 数据模型（核心表）
- `users`：id, uuid, is_guest, age_group, is_minor, parent_consent, created_at
- `word_levels`：id, code(L1–L4), name, description
- `words`：id, level_id, term, phonetic, audio_url, definition_zh, **image_type, image_url, association_text, accuracy_reviewed, status**（联想"须准确"：accuracy_reviewed 审核标记）
- `user_levels`：user_id, current_level, placement_done（分级/升降档结果）
- `study_records`：id, user_id, word_id, result, created_at
- `review_items`：user_id, word_id, **ease_factor, interval, repetitions, due_date**, last_reviewed（SM-2 字段）
- `chapters`（关卡）：id, code, title, level_id, word_ids, order
- `user_progress`：user_id, chapter_id, status, progress
- `user_points`：user_id, total_points, streak, last_checkin
- `checkins`：user_id, date
- `leaderboard_snapshots`：week, user_id, points, rank（周快照）
- `ad_events`：user_id, ad_type, action, is_minor, created_at

### 1.5 关键算法
**(1) 分级定档 + 升降档**
- 定档：分级测试 10–15 题自适应（答对变难、答错变易），按正确率映射到 L1–L4。
- 升降档信号：近 K 次测验正确率 **>90% → 升一档**；**<50% → 降一档**；用户可手动覆盖。

**(2) 间隔重复 SM-2（简化实现）**
```
输入: 复习质量 q (0-5), 旧 (EF, I, n)
if q < 3:
    n = 0; I = 1            // 重新学习
else:
    if n == 0: I = 1
    elif n == 1: I = 6
    else: I = round(I * EF)
    n = n + 1
EF = EF + (0.1 - (5 - q) * (0.08 + (5 - q) * 0.02))
EF = max(EF, 1.3)
due_date = now + I 天
```
- 服务端计算 + 客户端本地队列（离线可复习，联网同步）。

**(3) 周排行榜快照**
- 每周一 00:00 定时任务：按 `total_points` 排序生成上周快照存入 `leaderboard_snapshots`；前端拉 Top N + 用户周边排名，避免实时全量排序。

### 1.6 联想内容生产流水线（落地"趣味须准确"）
- 流程：模板化 + AI 生成 → **双校验（准确 + 趣味）** → 人工抽检 → 发布。
- 字段：`association_text`（联想文案）、`image_type/image_url`、`accuracy_reviewed`（是否过审）。
- 规范：联想区与"正确拼写/音标"视觉分离；文案不得出现错误拼写引导；敏感词过滤；未成年适龄。
- 三档图像：MVP Emoji/图标（零成本）→ 插画库 → AI 文生图（A1 验证后）。

### 1.7 广告与变现接入
- 激励视频：复活 / 提示 / 兑换"去广告体验券"。
- 插屏：关卡间隙（频次上限，如每关 ≤1 次、最小间隔）。
- **未成年分流**：age<18 或未确认成人 → 不投诱导/购物/社交类广告、降频。

### 1.8 合规与安全
- 年龄门（首启询问年龄/是否未成年）；未成年需家长同意（如收集信息）。
- 隐私政策、数据最小化、HTTPS、API 限流、内容审核（AI 图/文机审 + 人审）。
- 遵循个保法 / GDPR。

### 1.9 非功能需求
- 性能：首屏 <1.5s，动画 60fps，包体瘦身（按需懒加载）。
- PWA 离线学习；埋点 day 1；基础监控（错误/性能/广告疲劳）。

> 前后端并行；关键路径：B4(SM-2) → F5(复习) → V1(A/B)。

### 2.4 团队与假设
- **假设团队**：1 前端 + 1 后端 + 0.5 设计/内容 + 0.5 数据，可并行。
- 若仅 1 名全栈：墙钟拉长至 ~9–11 周，建议优先砍 V4 机动与 P0.2 精细度。
- 人天合计（MVP+验证）：前端 ~32d，后端 ~22d，内容/设计/数据 ~15d。

### 2.5 风险与缓冲
- **A1 不成立**（联想无显著提升）：A/B 早验证，不成立则回退"普通释义+轻量联想"，不阻塞上线。
- **广告伤体验（A3）**：设广告疲劳监控，超阈值自动降频。
- **未成年合规返工**：年龄门/家长同意尽早联调（V3），避免上线前卡审。
- **词库版权**：用自整理词表，避免爬付费词典。
